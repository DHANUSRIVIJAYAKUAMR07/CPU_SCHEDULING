First Come First Serve — Non-Preemptive

FCFS Scheduling

Algorithm Explanation

FCFS (First Come First Serve) is the simplest CPU scheduling algorithm. Processes are dispatched in the order of their arrival time in the ready queue — the process that arrives first gets the CPU first.

It is non-preemptive: once a process starts executing, it runs until completion. While easy to understand, it can cause the "Convoy Effect" where short processes wait behind long ones.

Shortest Job First — Preemptive / Non-Preemptive

SJF Scheduling

Algorithm Explanation

SJF selects the process with the smallest burst time from the ready queue. This minimizes the average waiting time and is provably optimal for batch processing scenarios.

Non-Preemptive SJF: Once a process starts, it runs to completion. Preemptive SJF (SRTF): If a new process arrives with a shorter remaining burst time, it preempts the running process.

Priority Scheduling

Algorithm Explanation

Each process is assigned a priority number. The CPU is allocated to the process with the highest priority (lowest number = highest priority). Equal priorities are resolved by arrival order.

The main drawback is starvation — low priority processes may wait indefinitely. This can be addressed with aging (gradually increasing priority of waiting processes).

Round Robin — Preemptive

Round Robin Scheduling

Algorithm Explanation

Round Robin assigns a fixed time quantum to each process in a cyclic manner. Each process runs for at most one quantum before yielding the CPU to the next process. It is inherently preemptive and designed for fairness.

The choice of time quantum is critical: too small increases context-switching overhead; too large degrades to FCFS behavior. Typical values range from 10–100ms in real systems.
